Ferriscope

Basic Usage

Command Line Options

USAGE:
    ferriscope [OPTIONS]

OPTIONS:
    -i, --interface <INTERFACE>    Network interface to capture from
    -f, --filter <FILTER>         Filter expression (tcpdump syntax)
    -o, --output <FILE>           Output file for packet capture
    -l, --list                    List available network interfaces
    -h, --help                    Print help information
    -V, --version                 Print version information

Filtering

Basic Filter Syntax

Filters use tcpdump/libpcap syntax. Here are common patterns:

Protocol Filters

# TCP only
ferriscope -i eth0 -f "tcp"

# UDP only
ferriscope -i eth0 -f "udp"

# ICMP only
ferriscope -i eth0 -f "icmp"

Port Filters

# Single port
ferriscope -i eth0 -f "port 80"

# Multiple ports
ferriscope -i eth0 -f "port 80 or port 443"

# Port ranges
ferriscope -i eth0 -f "portrange 1-1024"

Host Filters

# Single host
ferriscope -i eth0 -f "host 192.168.1.1"

# Source host
ferriscope -i eth0 -f "src host 192.168.1.1"

# Destination host
ferriscope -i eth0 -f "dst host 192.168.1.1"

Combining Filters

Use logical operators to combine filters:

# AND operator
ferriscope -i eth0 -f "tcp and port 80"

# OR operator
ferriscope -i eth0 -f "port 80 or port 443"

# NOT operator
ferriscope -i eth0 -f "not port 22"

Advanced Filters

TCP Flags

# SYN packets
ferriscope -i eth0 -f "tcp[tcpflags] & tcp-syn != 0"

# SYN-ACK packets
ferriscope -i eth0 -f "tcp[tcpflags] & (tcp-syn|tcp-ack) == (tcp-syn|tcp-ack)"

Packet Size

# Packets larger than 1000 bytes
ferriscope -i eth0 -f "greater 1000"

# Packets smaller than 100 bytes
ferriscope -i eth0 -f "less 100"

Platform-Specific Notes

Linux

Installation

# Debian/Ubuntu
sudo apt-get update
sudo apt-get install libpcap-dev

# Fedora
sudo dnf install libpcap-devel

# Arch Linux
sudo pacman -S libpcap

Permissions Setup

  1. Temporary (using sudo):
sudo ferriscope
  1. Permanent (recommended):
# Set capabilities
sudo setcap cap_net_raw,cap_net_admin=eip target/release/ferriscope

# Verify capabilities
getcap target/release/ferriscope
  1. Using network group (alternative):
# Add user to network group
sudo usermod -a -G network $USER

# Set group permissions
sudo chgrp network target/release/ferriscope
sudo chmod 750 target/release/ferriscope

Interface Names

  • wlan0, wlan1: Wireless interfaces
  • enp0s3: Predictable network interface names
  • lo: Loopback interface

System Configuration

# Increase capture buffer size

echo "net.core.rmem_default=2097152" | sudo tee -a /etc/sysctl.conf

Performance Tuning

Capture Buffer Size

The capture buffer size affects how many packets can be queued before processing:

# Increase buffer size on Linux
sudo sysctl -w net.core.rmem_max=2097152
sudo sysctl -w net.core.rmem_default=2097152

Filter Optimization

Efficient Filters

# Good - specific and focused
ferriscope -i eth0 -f "tcp port 80"

# Better - using port ranges
ferriscope -i eth0 -f "tcp portrange 80-443"

# Avoid - too broad
ferriscope -i eth0 -f "ip"

Filter Order

Place the most selective filters first:

# Good
ferriscope -i eth0 -f "port 80 and tcp"

# Less efficient
ferriscope -i eth0 -f "tcp and port 80"

Memory Usage

  • Use specific filters to reduce packet capture volume
  • Clear packet history periodically using 'c'
  • Save to file instead of keeping in memory for long captures

CPU Usage

Factors affecting CPU usage:

  1. Capture filter complexity
  2. Packet rate
  3. Interface speed
  4. Display update frequency

Tips

  • Use simple filters when possible
  • Capture on specific ports/protocols
  • Avoid capturing all packets without filters
  • Use output files for high-volume captures

Troubleshooting

Common Issues

Permission Denied

Error: Failed to open capture device 'eth0'. This usually means insufficient permissions.

Solution:

# Option 1: Run with sudo
sudo ferriscope

# Option 2: Set capabilities (Linux only)
sudo setcap cap_net_raw,cap_net_admin=eip target/release/ferriscope

Interface Not Found

Error: Device not found

Solution:

  1. List available interfaces:
ferriscope -l
  1. Use the correct interface name
  2. Check if interface is up:
# Linux
ip link show

# macOS
ifconfig

# Windows
ipconfig

Invalid Filter Syntax

Error: Invalid filter expression

Solution:

  1. Check filter syntax
  2. Use quotes around complex filters
  3. Refer to the Filter Syntax guide

Performance Issues

High CPU Usage

Symptoms:

  • System slowdown
  • Delayed packet display
  • UI lag

Solutions:

  1. Use more specific filters
  2. Reduce capture buffer size
  3. Clear packet history regularly
  4. Save to file instead of keeping in memory

Memory Issues

Symptoms:

  • Increasing memory usage
  • Slow response time
  • System warnings

Solutions:

  1. Clear packet history (c key)
  2. Use output files for long captures
  3. Apply more specific filters
  4. Limit capture duration

UI Issues

Terminal Display Problems

Symptoms:

  • Garbled output
  • Missing UI elements
  • Incorrect colors

Solutions:

  1. Check terminal size:
# Should be at least 80x24
stty size
  1. Verify terminal supports colors:
echo $TERM
  1. Reset terminal:
reset

Crash Recovery

If the program crashes, restore terminal:

# Reset terminal
reset

# Or
stty sane
